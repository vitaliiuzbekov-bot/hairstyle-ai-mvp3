# Текущее состояние проекта (НейроСтилист AI)

## Что было сделано:
1. **Биллинг и Ошибка Авторизации**: Исправлена логика в `billing.ts`. Ранее пользователи, у которых в базе не был записан `tgId` (созданные до введения обязательной привязки к Telegram), получали `AUTHORIZATION_ERROR`, из-за чего интерфейс писал "не хватает кредитов/звезд" даже если на балансе было 999. Теперь старым пользователям автоматически прописывается `tgId`.
2. **Ежедневный Бонус**: Исправлен баг, из-за которого ежедневный бонус (+1 генерация) добавлялся только визуально в локальное хранилище браузера, но не синхронизировался с сервером Firebase. Теперь сервер получает токен.
3. **Камера в Telegram**: В `usePhotoHandlers.ts` убрана жесткая проверка мобильного устройства в Telegram, из-за которой кнопка "Сделать фото" принудительно открывала файловый менеджер (который на iOS мог открывать только галерею). Теперь приложение сначала пытается запросить встроенную камеру через `navigator.mediaDevices.getUserMedia`.
4. **Фотографии Топ Стрижек**: Решена проблема с неотображением иконок топовых стрижек после экспорта. Ранее они зависели от локальных файлов `m_french_crop.jpg` и т.д., которые вырезались при определенной конфигурации сборки Vite. Компонент `PresetsCarousel` полностью переведен на использование `LazyImage`, который запрашивает картинки из официальной библиотеки через серверный API. Названия стрижек (например, "French Crop") жестко синхронизированы с базой (пункт 8), поэтому загрузка происходит из кэша мгновенно.

5. **Экспорт в PDF**: Картинка исходного фото теперь загружается через специальный прокси (`/api/proxy-image`) для обхода CORS ограничений, поэтому она гарантированно попадает в PDF-файл.

6. **Ошибки старых версий Telegram**: В компоненте добавления на главный экран (`PWAPrompt`) добавлена строгая проверка версии Telegram (`tg.isVersionAtLeast("8.0")`). Это предотвращает падение приложения (unsupported method error) у пользователей с устаревшими клиентами Telegram, где функция `checkHomeScreenStatus` отсутствует.

7. **Исправление ошибок загрузки чанков**: Добавлен авторефреш при ошибке загрузки динамических чанков, которые могут возникать после деплоя новых версий.

8. **Исправление Nginx 403 HTML**: Убраны ответы с 403 кодом, которые перехватывались nginx, что приводило к ошибке парсинга JSON на клиенте. Теперь сервер отдает 400 Bad Request.

9. **Изображения референсов (Библиотека)**: Названия стрижек в карусели "Топ-стрижек месяца" (например, "Френч кроп") не совпадали с названиями в официальной базе библиотеки (там было "French Crop"). Из-за этого срабатывала защита сервера (возвращая 400/403). Имена полностью синхронизированы, теперь все картинки карусели загружаются корректно.

10. **Баг с 999 генерациями**: У разработчиков в интерфейсе появлялось 999 генераций, но сервер выдавал ошибку "Закончились генерации". Причина заключалась в том, что Nginx-прокси вырезал кастомный заголовок `x-developer-mode`. Теперь статус разработчика передается прямо в теле запроса (FormData), обходя ограничения прокси.

11. **Telegram Stories**: Исправлена ошибка "К сожалению, не удалось поделиться в сторис". Проблема была в том, что функция браузера для генерации видео (`canvas.captureStream`) не поддерживается на некоторых версиях iOS/Safari в Telegram. Теперь добавлен безопасный "фолбэк": если создание видео падает с ошибкой, приложение мгновенно создает и отправляет в сторис статичную фотографию в формате До/После (коллаж).

12. **Качество генерации**: Улучшен промпт Gemini (системные инструкции) для предотвращения изменения черт лица у лысых или бритоголовых людей при кардинальной смене прически. Теперь нейросеть получает строгие инструкции не менять изначальное строение головы.

13. **Тестирование Stories**: Улучшена логика предупреждений для тестирования функции "В сторис" вне среды Telegram (например, в песочнице Google Studio). Если кнопка нажимается из обычного браузера, пользователю показывается понятное окно с объяснением.

14. **Сжатие фотографий на клиенте (Оптимизация)**: Параметры сжатия увеличены до 1024x1024 пикселей (качество 80%), как и требовалось. Добавлено принудительное масштабирование кадра (до 1024px) прямо в момент снимка с веб-камеры (`useCamera.ts`), что экономит оперативную память и предотвращает лаги при снимках с 4K-камер телефонов.

## Результаты предрелизной проверки (Pre-Export Validation)
Перед выгрузкой в GitHub был проведен детальный аудит приложения:
* **Успешная сборка (Vite + esbuild)**: Команда `npm run build` отрабатывает без ошибок, `server.cjs` компилируется корректно, статика бандлится.
* **Карусель пресетов**: Проверена логика загрузки изображений в `PresetsCarousel.tsx`. Проблема с пропажей картинок решена за счет использования API-загрузчика (LazyImage) с идеальным совпадением имен из глобального кэша `haircutLibrary.ts`.
* **API Прокси для PDF**: Интеграция `pdfExport.ts` и `generateRouter` для `proxy-image` настроена на консистентные URL (`/api/proxy-image`).
* **Клейм Ежедневного Бонуса**: Проверена архитектура — клик в `DailyRewardModal` правильно диспатчит событие `daily_reward_claimed`, которое перехватывается в `useTokenManager` и надежно отправляет запрос на сервер (Firestore `updateDoc`).
* **Защита старых API**: Защитные проверки версии Telegram API (8.0+) работают и предотвращают краш WebApp SDK.

Приложение полностью стабильно и готово к финальному экспорту (Deploy).

15. **Безопасность биллинга (Fail-Closed)**: Устранена уязвимость (Fail-Open), при которой ошибки подключения к Firebase Admin SDK (отсутствие БД или `PERMISSION_DENIED`) возвращали `{ ok: true }`. Разрешены только домены из белого списка (`fal.media`, `v3.fal.media`, `storage.yandexcloud.net`, `firebasestorage.googleapis.com`, `images.unsplash.com`) и только протокол HTTPS. Это предотвращает использование сервера в качестве открытого прокси.
17. **Защита вебхука Telegram**: Эндпоинт `/api/set-telegram-webhook` в `src/server/routes/auth.ts` защищен обязательной проверкой заголовка `x-admin-secret`. Теперь для его вызова значение заголовка должно совпадать с переменной окружения `ADMIN_SETUP_SECRET`, иначе возвращается 403.
18. **Защита Developer Mode**: В эндпоинтах генерации (`generate.ts` и `reference.ts`) убрана проверка простого заголовка `x-developer-mode` (который мог подделать любой). Теперь используется криптографическая проверка через новый хелпер `isAuthorizedDeveloper` (в `tgAuth.ts`). Он проверяет валидность данных `x-telegram-init-data` (созданных Telegram) и сверяет Telegram ID пользователя со списком разрешенных из переменной `DEV_TELEGRAM_USER_IDS`.
19. **Безопасность Storage (Загрузка фото)**: Обновлены правила Firebase Storage (`storage.rules`). В директории `/uploads/` устранено разрешение `allow write: if true;`, которое позволяло любому желающему загружать любые файлы. Теперь загрузка разрешена только авторизованным пользователям (`request.auth != null`), только изображений (`image/.*`), и размер ограничен 15 МБ.
